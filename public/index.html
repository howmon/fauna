About
Fauna v1.0.0
Multi-provider AI assistant — use GitHub Copilot, OpenAI, Anthropic, or Google models.

Model routing:
• Models from API Keys → sent directly to that provider
• All other models → routed through GitHub Copilot

GitHub token priority:
1. Saved PAT   2. GH_TOKEN env   3. gh auth token   4. Keychain

API key models are fetched dynamically from each provider.

Chat

Type a message in the input box and press Enter to send. Press Shift+Enter for a newline.

Attach files by dragging them onto the chat window or clicking the attachment icon in the toolbar. Paste a file path or image directly from the clipboard. Click Add URL to attach web page content as context.

The context meter (bottom of input) shows how much of the model's context window the conversation is using. When it fills, older messages are automatically trimmed.

Model selection

Choose the active model from the dropdown in the top bar. The list is populated from your account subscription.

Agents & Agent Chips

Agents are AI personas with custom system prompts, tools, and permissions. Type @agent-name in the input box to activate one. An autocomplete dropdown appears as you type.

Once an agent is active, a badge appears in the top bar showing its name. Press Esc (with an empty input) to deactivate the current agent.

Agent Store

Open the Agent Store (Ctrl/Cmd+Shift+P) to browse, install, and manage agents from the registry. Installed agents appear as chips in the sidebar.

Create an agent from chat

Ask the AI to "create an agent for…" and it will emit a structured agent definition. An Open in Agent Builder card will appear in the response — click it to pre-fill the builder with the generated config.

Agent Builder

Open the builder with Ctrl/Cmd+Shift+B or from the Agent Store. The builder lets you create and edit agents with:

  • Name & icon — display name and emoji icon shown in chips and store
  • Description — short summary shown in autocomplete
  • System prompt — full instructions given to the AI when the agent is active
  • Permissions — which tools the agent can use (shell, browser, Figma, file write)
  • Model override — force a specific model for this agent
  • Shared prompt — (orchestrators only) context injected into every sub-agent

AI Generate

Click Generate with AI in the builder to describe what you want and have the model fill in the name, description, system prompt, and permissions automatically.

Orchestration & Delegation

An orchestrator agent coordinates a team of sub-agents. When the orchestrator runs, it emits delegation blocks that route subtasks to named sub-agents, each with its own system prompt and tools.

Sub-agent calls appear in a delegation panel below the orchestrator's response. Each delegate runs sequentially; results are fed back into context.

Stopping a delegation

Click the Stop button that appears in the delegation header while agents are running. Any in-progress sub-agent is cancelled and subsequent delegates are skipped.

Shared prompt

Content from the agent's Shared Prompt field is automatically prepended to every sub-agent's system prompt — useful for project context, shared conventions, or component docs.

Shell Execution

When the AI writes a bash code block, it renders as a runnable shell widget with a Run button. Output is displayed inline.

Auto-run mode

Enable Auto-run shell commands in Settings to execute bash blocks immediately without clicking Run. A badge on each block indicates it will run automatically.

Feed to AI

After a command runs, click Feed to AI to send the output back into the conversation so the model can continue working.

Stopping a command

A running command shows a Kill button. Click it to terminate the process.

File Operations

The AI has three structured block types for writing files — no shell required:

write-file
Creates or overwrites a file at the given path. Relative paths resolve against the conversation workspace under ~/.copilot/workspaces/.
append-file
Appends content to an existing file without overwriting.
replace-string
Targeted find-and-replace inside an existing file. Includes a SEARCH block (exact text to find) and a REPLACE block. Safer than full rewrites.
apply-patch
Applies a unified diff across one or more files — suitable for multi-file changes or renames.

Each block renders a confirmation widget. Click Write to apply, or Diff to preview changes first.

Artifacts

The artifact pane (right side) previews rich content produced by the AI:

  • HTML — rendered in a sandboxed iframe; interactive dashboards, tools, games
  • SVG — rendered as vector graphics
  • Image — screenshots or generated images produced by shell commands
  • Markdown — formatted document preview
  • JSON / CSV — structured data viewer
  • Code — syntax-highlighted source

Each artifact appears as a card in the chat; click Open to show it in the pane. Tabs at the top let you switch between multiple artifacts.

Browser Panel

The built-in browser panel lets the AI — and you — navigate any URL without leaving the app. It uses a full Electron WebView with a realistic browser user-agent.

Each conversation has its own set of browser tabs. The AI can open URLs, read page content, click links, fill forms, and take screenshots via tool calls.

You can also type in the URL bar manually and browse independently of the active conversation.

Figma Integration

Two integration modes are available:

Figma MCP (recommended)
Connects via the Model Context Protocol. The AI gets live read and write access to your Figma file — components, layers, variables, and more. Enable it by toggling the Figma MCP section in the sidebar.
FaunaMCP Plugin
A Figma plugin bundled with this app that runs a local relay. The AI writes figma-exec blocks containing Figma Plugin API JavaScript; the plugin executes them inside Figma. Use Figma Plugin Setup (plug icon in sidebar) to install it.

Figma Rules

Add rules (e.g. "always use design system tokens") that are injected into the AI's context whenever Figma is connected. Access them via the Figma section header in the sidebar.

Playbook

The Playbook (notebook icon in the sidebar footer) stores Learned Instructions — short rules the AI has discovered from successful approaches.

Entries are auto-saved when the AI identifies a reusable pattern, or you can add them manually. All enabled entries are injected into the system prompt on every request.

Toggle individual entries on/off with the checkbox. Delete entries with the trash icon. Dragging is not required — use the toggle to temporarily suppress an instruction without deleting it.

Spec Memory

The Spec Memory tab (inside Playbook) organises knowledge into categories and groups. Each group is a block of text that gets injected into the AI's system prompt when enabled.

Use it to store component specs, API documentation, design system rules, or any reusable reference the AI should know about. Categories can be exported and imported as JSON.

  • Click New Category to create a top-level namespace
  • Add groups inside a category to hold specific knowledge blocks
  • Toggle a group on/off to include or exclude it from the system context
  • Click Reset Built-in to restore the default categories without affecting custom ones

Agent Rules

Agent Rules ( in the sidebar footer) are hard constraints injected into every AI message, regardless of which agent or model is active.

Use them for global formatting preferences, safety guardrails, or behavioral overrides that should always apply, such as:

  • Always write complete commands — never leave code blocks empty
  • Reply in a specific language
  • Never include caveats or disclaimers

Rules are persisted locally and survive app restarts.

Settings

Open Settings from the bottom of the sidebar ( Settings button).

Personal Access Token
Override the auto-detected Copilot token. Required if you are not signed into the gh CLI. Generate one at github.com/settings/tokens with the copilot scope.
Thinking Budget
Controls extended thinking for supported models (Claude, o-series). Higher budgets allow deeper analysis at the cost of more tokens: Off / Low (~1K) / Medium (~5K) / High (~10K) / Max (~32K).
Maximum Context Turns
How many past conversation messages to include in each request. Higher values increase accuracy for long tasks but also cost more tokens.
Auto-run Shell Commands
When enabled, bash blocks execute automatically without a click. Use with caution.
System Prompt
Override the default system prompt for all conversations. Changes take effect on the next message.
